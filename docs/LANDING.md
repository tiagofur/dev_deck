# Landing Page Structure

This document outlines the layout and sections for the **devdeck.ai** landing page.

[Leer en español](LANDING.es.md)

---

## 1. Hero Section
- **Headline:** Bold and direct. Focus on the "external memory" concept.
- **Subheadline:** Explain the value (organize repos, CLIs, snippets with AI).
- **CTA:** "Download Desktop" (Primary) / "Open Web App" (Secondary).
- **Visual:** High-quality screenshot or short video loop of the "Global Capture" flow.

---

## 2. The Problem (Agitate)
A grid of 3 common dev frustrations:
1. *"I saw a tool on X but forgot to save it."*
2. *"I have 2000 GitHub stars and can't find anything."*
3. *"Searching my browser bookmarks is a black hole."*

---

## 3. Features (Solve)

### 3.1 One-Click Capture
- Browser extension.
- Terminal CLI.
- Smart Paste (Clipboard detection).

### 3.2 AI Brain
- Auto-summaries.
- Intent-based semantic search.
- Smart tagging.

### 3.3 Built for Developers
- Neo-brutalist Dark Mode.
- Offline-first (Desktop).
- Open Source.

---

## 4. How it Works
1. **Capture:** Save from anywhere.
2. **Enrich:** AI summarizes and tags automatically.
3. **Find:** Search by intent, not just keywords.

---

## 5. Pricing / Open Source
- **Community:** Free, Open Source, Self-hostable.
- **Pro (Future):** Cloud sync, Team collaboration, Premium AI models.

---

## 6. Footer
- Links to GitHub, Documentation, Roadmap.
- "Buy Me a Coffee" link.
- Social (X/Twitter).
